# データ構造スキーマ

ガントチャートビューアーが想定するデータ構造です。実データのサンプルは [`../examples/sample-data.json`](../examples/sample-data.json) を参照してください。

> 機械可読な **JSON Schema** も用意しています → [`schema.json`](schema.json)。生成した JSON を検証したり、AI に自己検証させる際に使えます（`examples/*.json` は検証通過を確認済み）。

すべての日付は `YYYY-MM-DD` 形式。`ja` / `en` は日英切替に対応するための2言語ラベルです（片方だけ運用する場合は同じ値でも可）。

---

## トップレベル

```
{
  "meta":      { "ja": {...}, "en": {...} },
  "calendar":  { "workweek": [1,2,3,4,5], "holidays": [...], "extraWorkdays": [...] },
  "assignees": { <担当コード>: Assignee, ... },
  "projects":  [ Project, ... ]
}
```

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `meta` | オブジェクト | 任意 | 左上のタイトル／サブタイトル（日英）。省略時は既定の文言。 |
| `calendar` | オブジェクト | 任意 | 稼働日カレンダー（稼働曜日・休業日・特別稼働日）。省略時は月〜金稼働・祝日なし。 |
| `assignees` | オブジェクト | 任意 | 担当者マスタ。タスクの `who` はこのキー（担当コード）を参照します。省略時も `who` の色/イニシャルは自動生成。 |
| `projects` | 配列 | **必須** | プロジェクトの一覧。上から順に縦に並びます。複数プロジェクトを1つの時間軸で俯瞰できます。空だと読込エラー。 |

## Calendar（稼働日カレンダー）

```
"calendar": {
  "workweek": [1, 2, 3, 4, 5],
  "holidays": [ { "date": "2026-07-20", "ja": "海の日", "en": "Marine Day" }, ... ],
  "extraWorkdays": ["2026-08-15"]
}
```

祝日／休業日は**利用者がJSON側で定義**します（ビューアーには内蔵しません）。日スケールで色分け表示されます。

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `workweek` | 数値配列 | 任意 | **稼働する曜日**（`0`=日, `1`=月 … `6`=土）。既定は `[1,2,3,4,5]`（月〜金）。**土日も稼働**にしたい場合は `[0,1,2,3,4,5,6]` のように含める。ここに無い曜日は「非稼働（週末）」としてグレー網掛け。 |
| `holidays` | 配列 | 任意 | 休業日。`{ "date", "ja", "en" }`（日付だけの文字列も可）。ローズ色で網掛け＋日付ヘッダー強調＋ホバーで名称表示。 |
| `extraWorkdays` | 文字列配列 | 任意 | 「本来は休みだが稼働する日」（休日出勤の土曜など）。指定日は網掛けを**解除**して稼働日扱い。 |

> 判定の優先順位：`extraWorkdays`（稼働）＞ `holidays`（休業）＞ `workweek` 外（週末）。表示は「表示レイヤー」の週末シェード／祝日でオン・オフできます。日付ヘッダーには曜日も表示されます。

## Meta（ヘッダー文言）

```
"meta": {
  "ja": { "title": "ガントチャート ビューアー", "subtitle": "ソフトウェア開発ロードマップ" },
  "en": { "title": "Gantt Chart Viewer",        "subtitle": "Software delivery roadmap" }
}
```

左上に表示されるタイトルとサブタイトル。日英切替に追従します。

## Assignee（担当者）

```
"SM": { "ja": "佐藤", "en": "Sato", "color": "#3b6fd4", "abbr": "SM" }
```

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| キー名 | 文字列 | **必須** | 担当コード（例 `SM`）。タスクの `who` はこのキーを参照。**氏名そのもの**（例 `"山田花子"`）をキーにしてもOK。 |
| `ja` / `en` | 文字列 | 推奨 | 表示名。省略時はキー名を表示に流用。 |
| `color` | 文字列(hex) | 任意 | アバターの色。省略時はキーから見分けやすい色を自動割り当て（決定的・グレーにはならない）。 |
| `abbr` | 文字列 | 任意 | アバターに出す2文字。省略時は自動生成（英名 `Taro Sato`→`TS`／和名 `山田花子`→`山田`／短いコード `SM`→`SM`）。 |

> `assignees` に定義していない `who`（氏名文字列など）をタスクに直接書いても、色とイニシャルは自動生成されます。

## Project（プロジェクト＝最上位グループ）

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | 文字列 | **必須** | 一意のID。 |
| `ja` / `en` | 文字列 | 推奨 | プロジェクト名。省略時は片方を流用（両方無いと空表示）。 |
| `accent` | 真偽 | 任意 | `true` で主役プロジェクトとして淡いアクセント背景になります。既定 `false`。 |
| `milestones` | 配列 | 任意 | マイルストーン（◇）。既定は無し。 |
| `phases` | 配列 | **必須** | 工程（フェーズ）の一覧。空だと行が描画されません。 |

## Milestone（マイルストーン）

```
{ "date": "2026-09-22", "ja": "v3.0 公開", "en": "v3.0 Launch", "rose": true }
```

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `date` | 文字列 | **必須** | 日付。プロジェクト行の該当位置に◇で表示。 |
| `ja` / `en` | 文字列 | 推奨 | ラベル。 |
| `rose` | 真偽 | 任意 | `true` でリリース等の重要マイルストーンを赤系で強調。既定 `false`。 |

## Phase（工程＝中間グループ）

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | 文字列 | **必須** | プロジェクト内で一意のID。 |
| `ja` / `en` | 文字列 | 推奨 | 工程名。省略時は片方を流用。 |
| `tasks` | 配列 | **必須** | タスクの一覧。空だと工程行が描画されません。 |

> 工程行のバー・工数・期間・進捗は、配下タスクから**自動集計**されます（手入力しません）。

## Task（タスク＝末端項目）

```
{
  "id": "3.2",
  "ja": "バックエンド実装", "en": "Backend build",
  "who": "KB",
  "plan":   { "start": "2026-06-25", "end": "2026-07-24" },
  "actual": { "start": "2026-06-26", "end": null },
  "progress": 0.55,
  "personDays": 22,
  "critical": true,
  "deps": ["3.1"]
}
```

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | 文字列 | **必須** | 一意のID。`deps` から参照されます。 |
| `ja` / `en` | 文字列 | 推奨 | タスク名。省略時は片方を流用。 |
| `who` | 文字列 | 任意 | 担当コード（`assignees` のキー）。省略時はアバターなし。 |
| `plan` | オブジェクト | **必須** | 予定期間 `{ start, end }`。ガントの「予定枠」に対応。無いとバーが描けません。 |
| `actual` | オブジェクト | 任意 | 実績期間 `{ start, end }`。「実績バー」に対応。省略時は未着手扱い（`{start:null, end:null}` と同義）。 |
| `progress` | 数値 0〜1 | 任意 | 進捗率（`0.55` = 55%）。イナズマ線とバー内の塗りに使用。既定 `0`。 |
| `personDays` | 数値 | 任意 | 工数（人日）。工程・プロジェクトへ集計されます。既定 `0`。 |
| `critical` | 真偽 | 任意 | クリティカルパス上のタスクなら `true`。既定 `false`。 |
| `deps` | 配列 | 任意 | 先行タスクの `id` 配列（依存の矢印を描画）。既定は無し。 |

### 予実（plan / actual）の状態表現

| 状態 | 表現 |
|---|---|
| 未着手 | `actual.start = null`（実績バーなし） |
| 進行中 | `actual.start` あり・`actual.end = null` |
| 完了 | `actual.end` あり・`progress = 1` |
| オーバーラン（遅延着地） | `actual.end` が `plan.end` より後 → はみ出し分を赤で表示 |

### 自動計算される表示要素（データに持たせない）

- 工程・プロジェクトの集計バー / 集計工数 / 集計進捗
- **イナズマ線**（各タスクの `progress` を予定期間上にマッピングして描画）
- **本日線**・現在日ハイライト（システム日付基準）
- ステータス色分け（順調 / 要注意 / 遅延 / 完了）※ `plan` `actual` `progress` と本日から判定

## 最小構成の例

```json
{
  "assignees": { "A": { "ja": "担当A", "en": "A", "color": "#3b6fd4" } },
  "projects": [
    {
      "id": "P1", "ja": "プロジェクト", "en": "Project", "accent": true,
      "milestones": [],
      "phases": [
        {
          "id": "1", "ja": "工程1", "en": "Phase 1",
          "tasks": [
            {
              "id": "1.1", "ja": "作業", "en": "Task", "who": "A",
              "plan":   { "start": "2026-07-01", "end": "2026-07-10" },
              "actual": { "start": "2026-07-01", "end": null },
              "progress": 0.3, "personDays": 5, "critical": false, "deps": []
            }
          ]
        }
      ]
    }
  ]
}
```
