# データ構造スキーマ（v2）

ガントチャートビューアーが想定するデータ構造です。実データのサンプルは [`../examples/sample-data.json`](../examples/sample-data.json) を参照してください。

すべての日付は `YYYY-MM-DD` 形式。`ja` / `en` は日英切替に対応するための2言語ラベルです（片方だけ運用する場合は同じ値でも可）。

> **v2 の変更点**: 階層は `children` による**再帰構造**になり、任意の深さまで採れます（プロジェクト→工程→タスク→3階層固定を廃止）。旧 v1 形式（`phases` / `tasks`）も引き続き**読み込み可能**（自動変換）。ビューアーの編集モードから保存すると常に v2 形式で書き出されます。

---

## トップレベル

```
{
  "meta":      { "ja": {...}, "en": {...} },
  "calendar":  { "workweek": [1,2,3,4,5], "holidays": [...], "extraWorkdays": [...] },
  "assignees": { <担当コード>: Assignee, ... },
  "projects":  [ Project, ... ]
}
```

| キー | 型 | 説明 |
|---|---|---|
| `meta` | オブジェクト | 左上のタイトル／サブタイトル（日英）。省略時は既定の文言。 |
| `calendar` | オブジェクト | 稼働日カレンダー（稼働曜日・休業日・特別稼働日）。省略時は月〜金稼働・祝日なし。 |
| `assignees` | オブジェクト | 担当者マスタ。タスクの `who` はこのキー（担当コード）を参照します。 |
| `projects` | 配列 | プロジェクトの一覧。上から順に縦に並びます。複数プロジェクトを1つの時間軸で俯瞰できます。 |

## Calendar（稼働日カレンダー）

```
"calendar": {
  "workweek": [1, 2, 3, 4, 5],
  "holidays": [ { "date": "2026-07-20", "ja": "海の日", "en": "Marine Day" }, ... ],
  "extraWorkdays": ["2026-08-15"]
}
```

祝日／休業日は**利用者がJSON側で定義**します（ビューアーには内蔵しません）。日スケールで色分け表示されます。

| キー | 型 | 説明 |
|---|---|---|
| `workweek` | 数値配列 | **稼働する曜日**（`0`=日, `1`=月 … `6`=土）。既定は `[1,2,3,4,5]`（月〜金）。**土日も稼働**にしたい場合は `[0,1,2,3,4,5,6]` のように含める。ここに無い曜日は「非稼働（週末）」としてグレー網掛け。 |
| `holidays` | 配列 | 休業日。`{ "date", "ja", "en" }`（日付だけの文字列も可）。ローズ色で網掛け＋日付ヘッダー強調＋ホバーで名称表示。 |
| `extraWorkdays` | 文字列配列 | 「本来は休みだが稼働する日」（休日出勤の土曜など）。指定日は網掛けを**解除**して稼働日扱い。 |

> 判定の優先順位：`extraWorkdays`（稼働）＞ `holidays`（休業）＞ `workweek` 外（週末）。表示は「表示レイヤー」の週末シェード／祝日でオン・オフできます。日付ヘッダーには曜日も表示されます。

## Meta（ヘッダー文言）

```
"meta": {
  "ja": { "title": "ガントチャート ビューアー", "subtitle": "ソフトウェア開発ロードマップ" },
  "en": { "title": "Gantt Chart Viewer",        "subtitle": "Software delivery roadmap" }
}
```

左上に表示されるタイトルとサブタイトル。日英切替に追従します。

## Assignee（担当者）

```
"SM": { "ja": "佐藤", "en": "Sato", "color": "#3b6fd4", "abbr": "SM" }
```

| キー | 型 | 説明 |
|---|---|---|
| キー名 | 文字列 | 担当コード（例 `SM`）。タスクの `who` はこのキーを参照。**氏名そのもの**（例 `"山田花子"`）をキーにしてもOK。 |
| `ja` / `en` | 文字列 | 表示名。 |
| `color` | 文字列(hex) | アバターの色。**省略可** — 省略時はキーから見分けやすい色を自動割り当て（決定的・グレーにはならない）。 |
| `abbr` | 文字列 | アバターに出す2文字。**省略可** — 省略時は自動生成（英名 `Taro Sato`→`TS`／和名 `山田花子`→`山田`／短いコード `SM`→`SM`）。 |

> `assignees` に定義していない `who`（氏名文字列など）をタスクに直接書いても、色とイニシャルは自動生成されます。

## Project（プロジェクト＝最上位グループ）

| キー | 型 | 説明 |
|---|---|---|
| `id` | 文字列 | 一意のID。 |
| `ja` / `en` | 文字列 | プロジェクト名。 |
| `accent` | 真偽 | `true` で主役プロジェクトとして淡いアクセント背景になります。 |
| `milestones` | 配列 | マイルストーン（◇）。省略可。 |
| `children` | 配列 | 配下の Node 一覧（下記）。v1 の `phases` でも読み込み可。 |

## Milestone（マイルストーン）

```
{ "date": "2026-09-22", "ja": "v3.0 公開", "en": "v3.0 Launch", "rose": true }
```

| キー | 型 | 説明 |
|---|---|---|
| `date` | 文字列 | 日付。その行の該当位置に◇で表示。 |
| `ja` / `en` | 文字列 | ラベル。 |
| `rose` | 真偽 | 省略可。`true` でリリース等の重要マイルストーンを赤系で強調。 |

> `milestones` はプロジェクトだけでなく、**任意の Node（グループ行）にも置けます**。工程ごとの中間ゲート（レビュー完了など）はそのグループ行に付けると見通しが良くなります。

## Node（タスク / グループ — 再帰）

プロジェクト配下の要素はすべて同じ Node 型です。**`children` が空でなければグループ（集計行）、空（または省略）ならタスク（末端）**として描画されます。深さは任意（1.1.1.1…といくらでも掘れる）。

> グループ行のバー・工数・期間・進捗は配下タスクから**自動集計**されます（グループに `plan` 等を手入力しません）。v1 の Phase は「`children` を持つ Node」として読み替えられます。

## Task（タスク＝末端の Node）

```
{
  "id": "3.2",
  "ja": "バックエンド実装", "en": "Backend build",
  "who": "KB",
  "plan":   { "start": "2026-06-25", "end": "2026-07-24" },
  "actual": { "start": "2026-06-26", "end": null },
  "progress": 0.55,
  "personDays": 22,
  "critical": true,
  "deps": ["3.1"],
  "note": "外部APIのレート制限に注意",
  "children": []
}
```

| キー | 型 | 説明 |
|---|---|---|
| `id` | 文字列 | 一意のID。`deps` から参照されます。 |
| `ja` / `en` | 文字列 | タスク名。 |
| `who` | 文字列 | 担当コード（`assignees` のキー）。 |
| `plan` | オブジェクト | 予定期間 `{ start, end }`。ガントの「予定枠」に対応。 |
| `actual` | オブジェクト | 実績期間 `{ start, end }`。「実績バー」に対応。 |
| `progress` | 数値 0〜1 | 進捗率（`0.55` = 55%）。イナズマ線とバー内の塗りに使用。 |
| `personDays` | 数値 | 工数（人日）。工程・プロジェクトへ集計されます。 |
| `critical` | 真偽 | クリティカルパス上のタスクなら `true`。 |
| `deps` | 配列 | 先行タスクの `id` 配列（依存の矢印を描画）。 |
| `note` | 文字列 | 備考。省略可。編集モードの詳細パネルで編集。 |
| `children` | 配列 | 子 Node。空/省略＝タスク、あり＝グループ（上記 Node 参照）。 |

### 予実（plan / actual）の状態表現

| 状態 | 表現 |
|---|---|
| 未着手 | `actual.start = null`（実績バーなし） |
| 進行中 | `actual.start` あり・`actual.end = null` |
| 完了 | `actual.end` あり・`progress = 1` |
| オーバーラン（遅延着地） | `actual.end` が `plan.end` より後 → はみ出し分を赤で表示 |

### 整合規則（AI 生成時に守ること）

データ上は独立したフィールドでも、意味的に連動するものがあります。矛盾したデータは表示（ステータス色・イナズマ線）を誤らせます。

- **`actual.end` に日付を入れたら `progress` は必ず `1`**。完了は2箇所（実績終了日と進捗率）で表現される。ビューアーは読込時に `actual.end` があれば `progress` を 1 に自動補正するが、生成データ側でも矛盾させないこと。
- **`critical: true` は `deps` で連結した一続きの鎖にする**。孤立した critical タスクはクリティカル「パス」にならない。
- **依存は FS（先行の終了 → 自分の開始）のみ**。ラグ（3日後に開始等）や SS/FF 型の依存は表現できない。必要なら日付側で調整する。
- **進捗のマッピングは暦日基準**。イナズマ線・遅延判定は予定期間を暦日で等分して進捗位置を求める（稼働日換算ではない）。連休を跨ぐタスクでは位置が実感とややずれることがある。
- **`personDays` は省略可**。グループ進捗の集計は人日加重が基本だが、全タスクで省略した場合は期間（日数）加重、それも取れない場合は単純平均に自動フォールバックする。

### 自動計算される表示要素（データに持たせない）

- 工程・プロジェクトの集計バー / 集計工数 / 集計進捗
- **イナズマ線**（各タスクの `progress` を予定期間上にマッピングして描画）
- **本日線**・現在日ハイライト（システム日付基準）
- ステータス色分け（順調 / 要注意 / 遅延 / 完了）※ `plan` `actual` `progress` と本日から判定

## 最小構成の例

```json
{
  "assignees": { "A": { "ja": "担当A", "en": "A", "color": "#3b6fd4" } },
  "projects": [
    {
      "id": "P1", "ja": "プロジェクト", "en": "Project", "accent": true,
      "milestones": [],
      "children": [
        {
          "id": "1", "ja": "工程1", "en": "Phase 1",
          "children": [
            {
              "id": "1.1", "ja": "作業", "en": "Task", "who": "A",
              "plan":   { "start": "2026-07-01", "end": "2026-07-10" },
              "actual": { "start": "2026-07-01", "end": null },
              "progress": 0.3, "personDays": 5, "critical": false, "deps": [], "children": []
            }
          ]
        }
      ]
    }
  ]
}
```
